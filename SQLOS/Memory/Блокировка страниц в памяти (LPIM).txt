Блокировка страниц в памяти ( LPIM ) используется для решения проблемы, которая может возникнуть между Windows и SQL Server.

Блокировка страниц в памяти гарантирует, что давление в памяти Windows не сможет лишить SQL Server ресурсов или перенести память SQL Server в файл подкачки Windows Server, что значительно снижает производительность.

Политика LPIM должна быть включена по умолчанию для SQL Server, при условии, что:
    - Сервер является физическим, а не виртуальным.
    - Физическая память превышает 16 ГБ (для ОС требуется собственный рабочий набор).
    - Максимальная память сервера была установлена ​​соответствующим образом (SQL Server не может использовать все, что он видит).
    - Счетчики производительности Memory\Available Mbytes контролируется регулярно (чтобы сохранить свободную память).
    
Побочным эффектом использования заблокированных страниц является то, 
что они требуют немного меньшего количества служебных данных для управления (потому что они не могут быть перемещены). 
Это может привести к реальному выигрышу в производительности на высокопроизводительных серверах SQL.

Если LPIM работает, в журнале ошибок SQL Server вы увидите следующее сообщение: Using Locked Pages in the Memory Manager.
