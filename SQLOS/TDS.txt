Сетевой интерфейс SQL Server (SQL Server Network Interface (SNI)) - это уровень протокола, который устанавливает сетевое соединение между клиентом и сервером.
На клиенте оператор упаковывается в пакет TDS с помощью сетевого интерфейса SQL Server и отправлен на уровень протокола на SQL Server, где он разварачивается, идентифицируется и отправляется в командный парсер.

TDS (Tabular Data Stream) - это протокол уровня приложения, который облегчает взаимодействие между клиентами и SQL Server.

TDS была реализована по протоколам TCP/IP, VIA (Virtual Interface Architecture) и протоколам Named Pipes (протокол VIA устарел и будет удален в будущей версии Microsoft SQL Server).

Вы также можете использовать протокол Shared Memory для подключения к экземпляру SQL Server, 
но его можно использовать только для подключения с того же компьютера, на котором работает движок базы данных.

Можно использовать sys.dm_exec_connections, чтобы увидеть физические подключения к серверу базы данных. 
Столбец net_transport, показывает физический транспортный протокол, используемый этим соединением (TCP, Shared memory, Session (когда соединение имеет несколько активных наборов результатов или MARS)).

TDS предоставляет несколько функций, включая аутентификацию и идентификацию, канальное шифрование, спецификацию запросов в SQL, массовые операции вставки, вызов хранимой процедуры или пользовательской функции и возврат данных.

ODBC, JDBC и OLE DB - это библиотеки, которые используют TDS для передачи данных между клиентом и SQL Server.
Запросы TDS и ответы между клиентом и SQL Server могут быть проверены с использованием различных инструментов, включая Microsoft Message Analyzer.

Клиент SQL Server может использовать TDS для выполнения следующих типов запросов при общении с сервером базы данных:
    - Пакет SQL.
    - Удаленный вызов процедуры.
    - Массовая загрузка.
    
Порт TCP по умолчанию, используемый механизмом базы данных, составляет 1433. SQL Server Browser использует порт UDP 1434.

Когда используется порт TCP/IP, отличный от значения по умолчанию, номер порта должен либо указываться в строке подключения, либо может быть разрешен службой SQL Server Browser, если она включена.

pipe\sql\query по умолчанию используется при настройке протокола Named Pipes.

Клиент может использовать следующие прикладные программные интерфейсы (API) или библиотеки для подключения к экземпляру SQL Server:
    - ODBC (Open Database Connectivity) - это открытый стандарт API, основанный на спецификациях интерфейса Call-Level Interface (CLI), первоначально разработанных Microsoft в начале 1990-х годов. 
      ODBC стал фактическим стандартом для доступа к данным как в реляционных, так и в нереляционных системах управления базами данных.
    - OLE DB (Online Linking and Embedding Database) - это API, разработанный Microsoft для доступа не только к базам данных на базе SQL, но и к другим источникам данных в единой форме.
      Несмотря на то, что OLE DB первоначально предназначалась как замена более высокого уровня для ODBC и ее преемник, ODBC остается наиболее широко используемым стандартом для доступа к базе данных.
    - JDBC (Java Database Connectivity) - это API, разработанный Sun Microsystems, который определяет, как клиент может обращаться к реляционной базе данных с языка программирования Java. 
      JDBC является частью платформы стандартного выпуска Java, которая с тех пор была приобретена корпорацией Oracle.
    - ADO.NET (объекты ActiveX Data Objects для .NET) представляет собой набор классов, которые предоставляют службы доступа к данным, является неотъемлемой частью среды Microsoft .NET.
      ADO.NET обычно используется для доступа к данным, хранящимся в системах реляционных баз данных, хотя он также может обращаться к нереляционным источникам.
