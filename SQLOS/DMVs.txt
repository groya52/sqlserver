sys.dm_os_schedulers
    Возвращает одну строку на планировщик. Каждый планировщик сопоставляется с отдельным процессором в SQL Server.
    
    parent_node_id
        Идентификатор узла, к которому принадлежит планировщик, также известный как родительский узел. Представляет узел NUMA.
    scheduler_id
        Идентификатор планировщика.
    cpu_id
        CPUID назначенный планировщику.
    is_online
        Если этот столбец возвращает 0, то планировщик не используется для обработки запросов или пакетов.
    current_tasks_count
        Количество текущих задач, связанных с этим планировщиком, включая следующие:
            - Задачи, ожидающие получения ресурсов.
            - Задачи, которые в настоящее время выполняются или которые могут выполняться, но ждут выполнения.
    runnable_tasks_count
        Количество задач, ожидающих выполнения в планировщике.
    current_workers_count
        Число рабочих потоков, связанных с этим планировщиком, включая рабочие потоки, не назначенных какой-либо задачи.
    active_workers_count
        Число рабочих потоков назначенных задаче.
    work_queue_count
        Количество задач, ожидающих рабочего потока.
    load_factor
        Внутреннее значение, указывающее воспринимаемую нагрузку на этот планировщик.
        Это полезно для целей отладки, когда планировщики выглядят неравномерно загруженными.
        Использование коэффициентов нагрузки помогает SQLOS лучше сбалансировать рабочую нагрузку.
        
sys.dm_os_workers
    Возвращает строку для каждого рабочего потока в системе.
    
    is_preemptive
        Значение 1 означает, что рабочий поток работает по расписанию с вытеснением. Любой рабочий поток, выполняющий внешний код, запускается по расписанию с вытеснением.
    is_fiber
        Значение 1 означает, что рабочий поток работает с использованием упрощенных пулов.
        
sys.dm_os_threads
    Возвращает список всех потоков SQLOS, работающих под управлением SQL Server.

    start_by_sqlserver
        Инициатор потока. 
            1 - SQL Server начал поток. 
            0 - другой компонент, например расширенная процедура из SQL Server, запустил поток.
    create_time
        Время создания потока.
    stack_bytes_used
        Количество байтов, активно используемых в потоке.
    affinity
        Маска CPU, на которой работает этот поток.
    locale
        Кэшированное значение идентификатора локали (LCID) для данного потока.
        
sys.dm_os_tasks
    Озвращает одну строку для каждой задачи, которая активна в экземпляре SQL Server.
    
    task_state
        Состояние задачи.
            
        PENDING
            Ожидание рабочего потока.
        RUNNABLE. 
            Готов к запуску, но ожидает получения кванта.
        RUNNING
            В настоящее время выполняется в планировщике.
        SUSPENDED.
            У задачи есть исполнитель, но ожидает ресурсов.
        DONE
            Завершено.
        SPINLOOP
            Обработка спин-блокировки, как при ожидании сигнала.

    context_switches_count
        Число переключений контекста планировщика, которые задача уже выполнила.
    pending_io_count
        Количество физических операций ввода-вывода, выполняемых этой задачей.
    pending_io_byte_count
        Общее количество байтов ввода-вывода, выполняемое этой задачей.
    pending_io_byte_average
        Среднее количество байтов ввода-вывода, выполняемое этой задачей.
    scheduler_id
        Идентификатор родительского планировщика.
    session_id
        Идентификатор сеанса, связанного с задачей.
        
sys.dm_os_waiting_tasks
    Возвращает сведения об очереди задач, ожидающих освобождения определенного ресурса.
    
    session_id
        Идентификатор сеанса, связанного с задачей.
    exec_context_id
        Идентификатор контекста выполнения, связанного с задачей.
    wait_duration_ms
        Общее время ожидания (в миллисекундах) для этого типа ожидания. Это время включает в себя signal_wait_time.
    wait_type
        Имя типа ожидания.
    resource_address
        Адрес ресурса, для которого выполняется задание.
    blocking_task_address
        Задача, в настоящее время хранящая этот ресурс.
    blocking_session_id
        Идентификатор сеанса блокирующей задачи.
    blocking_exec_context_id
        Идентификатор контекста выполнения задачи блокировки.
    resource_description
        Описание потребляемого ресурса.
