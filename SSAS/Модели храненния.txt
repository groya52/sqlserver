MOLAP
    Режим хранения по умолчанию. Требуется самое большое пространство для хранения.
    Хранилище сжимает данные и более эффективно распределяет их на диске, чем другие модели хранилищ, но для обработки данных также требуется больше времени.

    Когда SSAS обрабатывает данные для хранилища MOLAP, он использует DSV для генерации необходимых операторов SQL, которые извлекают данные из реляционного источника, 
    реструктурирует результаты для хранения на диске в проприетарном формате, а затем вычисляет и сохраняет скопления для мер (см. рис. Хранилище MOLAP.jpg). 
    
    Агрегации рассчитываются только в том случае, если вы определили правила агрегирования. 
    
    Когда SSAS получает запрос данных из запроса многомерного выражения (MDX), он получает агрегированные данные или подробные данные в хранилище MOLAP для подготовки результатов.

ROLAP
    SSAS не извлекает данные из реляционного источника во время обработки. Вместо этого обработка требует только проверки согласованности данных, следовательно, выполняется намного быстрее, чем обработка MOLAP.
    
    Агрегированные значения сохраняются в источнике данных.
    
    Когда SSAS получает запрос MDX для данных, он использует DSV для перевода запроса в оператор SQL и отправляет этот оператор в источник данных. Если источником данных является SQL Server, MDX переводится в T-SQL.
    
    Хранилище ROLAP - лучший выбор, когда вашим бизнес-пользователям требуется доступ к данным в режиме реального времени или когда таблица измерений содержит сотни миллионов строк.
    Потенциальным компромиссом является медленная производительность запросов из-за перевода запроса с MDX на SQL.

HOLAP
    Хранилище HOLAP представляет собой комбинацию MOLAP и ROLAP. Подробные данные хранятся в источнике реляционных данных, но агрегированные данные и индексы загружаются в хранилище MOLAP во время обработки.
    
    Запросы HOLAP потенциально медленнее для решения. 
    Требования к хранению на сервере SSAS не являются действительно более низкими, поскольку SSAS должен считывать все данные из источника данных для создания агрегатов и индексов и требует достаточно места для этого. 
    После этого SSAS располагает данными, что снижает требования к хранению. Поскольку вы должны учитывать достаточные накладные расходы на хранение, HOLAP не является хорошим выбором, если ваша цель - минимизировать хранение.
