Вы можете получить скрипт XMLA для своей базы данных, щелкнув правой кнопкой мыши по базе данных в окне « Проводник объектов» в Management Studio и перейдя в базу данных сценариев в качестве создания.
Затем этот скрипт может быть выполнен с сервером для создания базы данных и всех объектов в базе данных. Это то, что вы часто делаете для создания автоматизированных конфигураций развертывания, где сценарий выполняется как запланированные задания.

Второй вариант, который вы также можете использовать, - это Synchronizeкоманда что позволяет синхронизировать ваш сервер разработки или тестирования с производственной базой данных.
http://technet.microsoft.com/en-us/library/ms186658.aspx

Третий вариант, который обычно используется для развертывания проектов в производственных средах, используется Мастер развертывания служб Analysis Services . Это отдельный инструмент, установленный с SQL Server, который будет создавать файлы XMLA, которые могут быть выполнены позже на сервере, используя файлы, созданные средствами данных SQL Server на этапе сборки развертывания. Эти файлы хранятся в <project folder>\binпапке и состоят из.asdatabase, в.configsettings, с.deploymentoptions, и.deploymenttargetsфайлы.
.asdatabaseфайлсодержит информацию, необходимую для создания базы данных и всех объектов. .configsettingsфайлсодержит параметры развертывания, определенные в проекте. Это также включает в себя строку источника данных, а также параметры безопасности, используемые при подключении к источнику данных. .deploymentoptionsфайлсодержит параметры развертывания, такие как использование Transactional Deployment или обработка базы данных после развертывания. Наконец, .deploymenttargetsфайлыуказать, какой сервер будет использоваться во время развертывания.
Как пользователь, вы может запускать анализ Мастер развертывания служб в интерактивном режиме или из командной строки. Когда вы запускаете его в интерактивном режиме, вы можете использовать его для создания файла ответов, который позже может быть использован для развертывания на сервере. Использование командной строки может быть эффективным способом при планировании развертывания, чтобы он выполнялся на более позднем этапе.
http://msdn.microsoft.com/en-us/library/ms162758.aspx

Более подробную информацию о том, как настроить обработку и как ее можно контролировать, можно найти в Многомерном руководстве по производительности и операциям Microsoft Analysis Server Analysis Services, доступном по адресу http://download.microsoft.com/download/0/F/B/0FBFAA46- 2BFD-478F-8E56-7BF3C672DF9D / Microsoft% 20SQL% 20Server% 20Analysis% 20Services% 20Multidimensional% 20Performance% 20 и% 20Operations% 20Guide.pdf .

Более подробную информацию о обработке можно прочитать в техническом документе архитектуры обработки Analysis Services 2005, доступном по адресу http://msdn.microsoft.com/en-US/library/ms345142(v=SQL.90).aspx (хотя он написан для 2005 он по-прежнему актуальен для SQL Server 2012).

Иногда обработка даже на уровне раздела (покрытая позже) трудно вписаться в доступное окно пакета. В этом случае есть опция, которую вы можете использовать и которая имеет отдельный сервер обработки, а затем синхронизирует сервер запросов с сервером обработки. Подробную информацию об этом параметре можно найти в следующей статье из команды SQLCAT:
http://sqlcat.com/sqlcat/b/technicalnotes/archive/2008/03/16/analysis-services-synchronization-best-practices.asp

Более подробную информацию об инструментах, доступных для обработки, можно найти в статье «Выбрать инструмент или подход для обработки объектов служб Analysis Services» на странице http://msdn.microsoft.com/en-us/library/aa337509.aspx .


Рекомендуется использовать тот же ключ раздела в реляционной базе данных, что и в службах Analysis Services, поэтому SQL Server может оптимизировать обработку и не будет запрашивать все данные в таблице при выполнении задачи обработки. Для получения дополнительной информации о том, как создать секционированную таблицу в SQL Server, обратитесь к следующему разделу в «Электронная книга книг»:
http://msdn.microsoft.com/en-us/library/ms188730.aspx
