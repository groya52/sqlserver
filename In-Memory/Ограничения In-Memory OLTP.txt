Максимальный размер строки 8060 байт.
    Это ограничение не позволяет использовать ни один из типов данных LOB для объявления столбца (XML, CLR, пространственных типов данных или любого из типов данных MAX).
    
    Более эффективно использовать varbinary, а не nvarchar для хранения данных XML в случаях, когда вы не можете использовать тип данных XML.

    Когда нет возможно изменить типы данных, а необходимо хранить столбцы LOB в таблицах, есть несколько вариантов:
        Разделение данных между двумя таблицами, хранении ключевых атрибутов в таблицах оптимизированных для памяти и редко доступных атрибутах LOB в таблицах на диске. 
        
        Хранить данные LOB в таблицах оптимизированных для памяти, разделяя их на несколько 8000 байтовых блоков (см. https://www.safaribooksonline.com/library/view/expert-sql-server/9781484211366/9781484211373_Ch11.xhtml#list6).

Отсутсвие ограничения уникальности и внешних ключей (см. https://www.safaribooksonline.com/library/view/expert-sql-server/9781484211366/9781484211373_AppA.xhtml#Sec4).
    
Бинарная сортировка чувствительности к регистру для индексированных столбцов.
        
        
        
        
        
Ограничения:
- 
- Все таблицы оптимизированные для памяти должны иметь хотя бы один индекс. Не разрешается использование кучи.
- Триггеры DDL/DML не разрешены;
- Изменения схемы не допускаются (ALTER TABLE). Чтобы изменить схему таблицы, вам нужно будет удалить и повторно создать таблицу;
- Запрещается секционирование или сжатие таблицы оптимизированной для памяти;
- Когда используется свойство столбца IDENTITY, он должен быть инициализирован (1, 1).
