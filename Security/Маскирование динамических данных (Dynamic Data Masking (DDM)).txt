Маскирование динамических данных (Dynamic Data Masking) это технология, которая была внедрена в SQL Server 2016, 
что позволяет не-привилегированным пользователям видеть только подмножество атомного значения, хранящегося в кортеже.

Динамическое маскирование данных может быть реализовано с использованием синтаксиса MASKED WITH в CREATE TABLE или ALTER COLUMN.

Чтобы показать полную стоимость, пользователям должно быть предоставлено разрешение UNMASK.

Правило маскирования можно задать для столбца в таблице, чтобы замаскировать данные в этом столбце. 
Доступны четыре типа маскирования:
    default
        Когда вы используете эту функцию для маскирования данных, а пользователь с привилегиями только для чтения запрашивает эти данные, 
        механизм базы данных маскирует все значение и возвращает значение замены. Точный характер значения замены зависит от типа данных столбца.
    email
        Используется специально для маскировки адресов электронной почты.
    random
        Заменяет числовые значения случайным значением в пределах заданного диапазона.
    partial
        Позволяет точно указать, какие значения маскируются.
        
Вы не можете применять DDM к вычисляемым столбцам или зашифрованным столбцам.

Никакие данные не зашифровываются в покое или в движении. Механизм базы данных просто заменяет чувствительные данные неидентифицирующими символами.
