Насройка томов:
    - Том C. Только ОС.
    - Том E. Установочные файлы SQL Server, файлы данных SQL Server.
    - Том F. Файлы журнала SQL Server.
    - Том G. SQL Server TempDB данные и файлы журналов.
    - Том H. Резервные копии SQL Server.
    - Используйте дополнительные тома для самых больших файлов данных (более 2 ТБ) для управления хранением.
    - Используйте дополнительный том для наиболее активных баз данных и их файлов журналов.
    - Используйте дополнительный объем для больших объемов данных FILESTREAM.
    - Использование дополнительного тома для файлов с крупными реплицируемыми базами данных.
    
Параметры, которые необходимо учитывать для томов:
    - Таблица разделов GUID (GPT)
      GPT поддерживают файлы размером более 2 ТБ, 
      тогда как более старый тип привода MBR ограничен 2 ТБ.
    - Размер размещения файлового блока для томов SQL Server 64 КБ.
      
      Для исправления размера размещения файлового блока требуется форматирование диска.
      
      Cоздать новый том с соответствующим размером распределения файлов, 
      а затем переместить файлы на новый том.
      
      Fsutil fsinfo ntfsinfo <RootPath>
        Размер распределения файлового блока возвращается с помощью Bytes Per Cluster.

TempDB        
    Наилучшее количество файлов данных TempDB почти наверняка больше одного и меньше или равно количеству логических процессорных ядер. 
    Добавление слишком большого количества файлов данных TempDB может серьезно ухудшить производительность SQL Server.
