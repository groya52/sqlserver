Поддерживаемые типы данных char, varchar, nchar, nvarchar, text, ntext, image, xml, varbinary и varbinary(max).

При создании индекса FTS сначала должен быть создан полнотекстовый каталог (ссылка на полнотекстовый индекс).
Сам каталог является виртуальным объектом и значительно повышает производительность за счет устранения узких мест ввода-вывода. Каталог содержит все свойства, доступные для поиска.

Один полнотекстовый каталог может содержать несколько полнотекстовых индексов, но любой полнотекстовый индекс может быть частью только одного полнотекстового каталога.
Каждая база данных может содержать ноль или более полнотекстовых каталогов.

Чтобы создать новый полнотекстовый каталог, используется синтаксис CREATE FULLTEXT CATALOG (см. CREATE FULLTEXT CATALOG.sql).

Функция FULLTEXTCATALOGPROPERTY возвращает сведения о свойствах полнотекстовых каталогов (см. FULLTEXTCATALOGPROPERTY.sql).

Чтобы создать создать полнотекстовый индекс, используется синтаксис CREATE FULLTEXT INDEX (см. CREATE FULLTEXT INDEX.sql).

Чтобы создать пользовательский StopList, используйте синтаксис CREATE FULLTEXT STOPLIST (см. CREATE FULLTEXT STOPLIST.sql).
