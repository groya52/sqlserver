HOLDLOCK
    Эквивалентен подсказке SERIALIZABLE.
UPDLOCK
    Заставляет SQL Server принимать блокировки обновлений (U) вместо общих (S) блокировок при чтении таблицы и удерживать их до конца транзакции. 
    
    Использование блокировок обновлений может быть важным методом устранения блокировок переходов.
TABLOCK
    Блокировка применяется на уровне таблицы.
    
    Использование этой подсказки с инструкцией DELETE в куче позволяет SQL Server освобождать страницы по мере удаления строк.
PAGLOCK
    Блокировка применяется на уровне страниц.
    
    Чтобы запросить эксклюзивную (X) блокировку страницы, вы должны использовать подсказку XLOCK с подсказкой PAGLOCK.
TABLOCKX
    Применяется эксклюзивная (X) блокировка на уровне таблицы, которая хранится до конца транзакции. 
    
    Все эксклюзивные (X) блокировки сохраняются до конца транзакции независимо от уровня изоляции. Эта подсказка имеет тот же эффект, что и вместе с подсказками TABLOCK и XLOCK.
ROWLOCK
    Указывает, что вместо блокировки страниц или таблиц применяются блокировки строк. 
    
    При указании блокировок строк в транзакциях, выполняемых на уровне изоляции SNAPSHOT, они применяются только в случае, 
    когда подсказка ROWLOCK используется в сочетании с другими табличными подсказками, требующими блокировки, например UPDLOCK или HOLDLOCK.
READUNCOMMITTED | REPEATABLEREAD | SERIALIZABLE
    SQL Server должен использовать те же механизмы блокировки, что и при соответствующих уровнях изоляции.
READCOMMITTED
    Указывает, что операции чтения соответствуют правилам для уровня изоляции READ COMMITTED или READ COMMITTED SNAPSHOT.
READCOMMITTEDLOCK
    Указывает, что операции чтения соответствуют правилам для уровня изоляции READ COMMITTED путем использования блокировки,
    вне зависимости от значения параметра базы данных READ_COMMITTED_SNAPSHOT.
NOLOCK
    Равнозначен подсказке READUNCOMMITTED.
READPAST
    Заблокированные строки пропускаются. 
    
    Применяется только к транзакциям, работающим на уровне изоляции READ COMMITTED, и считывает только блокировки на уровне строк.
XLOCK
    SQL Server использует исключительную блокировку, которая хранится до конца транзакции по всем данным, обрабатываемым оператором.
    
    Эта блокировка может быть задана с помощью PAGLOCK или TABLOCK, и в этом случае исключительная блокировка применяется к указанному ресурсу.
